Code References

1. What did you look up?

For this assignment, I researched the CSS property border-radius. I chose this property because while it seems simple—it just rounds the corners of an element—it actually has complex syntax options for creating elliptical shapes and asymmetrical corners that I wanted to understand better.

2. What sites did you consult?

  • MDN Web Docs (Mozilla)
  • W3Schools
  • CSS-Tricks
  • GeeksforGeeks

3. What are the differences in the information presented?

The information varied significantly in depth. W3Schools focused entirely on the basics, providing a "Try it Yourself" editor that lets you see the code in action immediately. It was very beginner-friendly but lacked deep technical details.

In contrast, MDN Web Docs provided a highly technical breakdown of the syntax, including formal grammar and browser compatibility charts. CSS-Tricks offered the most visual approach; they used diagrams to explain how the "slash" syntax works to create elliptical corners, which the other sites barely touched on. GeeksforGeeks felt similar to W3Schools but included more written definitions rather than interactive tools.

4. What features should a code reference page offer?

I believe a great code reference page needs three specific features. First, it must have a copy-pasteable syntax block at the very top. Second, it needs a "playground" or live preview where I can mess up the code without breaking my own site. Finally, it needs a browser support table so I know if the code will work on older devices.

5. Reflection on the Resources

CSS-Tricks stood out as the best resource for learning how the property works visually. However, MDN remains the best for verifying technical accuracy. I found GeeksforGeeks the least helpful as the layout was cluttered with ads and the examples were less distinct than W3Schools.
